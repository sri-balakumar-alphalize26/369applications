Attendance & Late Tracking

Office hours, late detection & deductions, grace, waivers and the office timezone.

1. The flow

Check in → convert to Office Timezone → which session / expected start → late? (threshold) → which late # this month → past grace? → slab/hourly deduction → reason → waiver? → month-end report → final salary.

Every check-in is stored in UTC; the server location does not matter.

Working Days & Half-Day Fridays

Working days = checked weekdays (minus public holidays). A half-day Friday (a Friday in a configured position, e.g. 2,4,5) counts as 0.5, even when Friday is unchecked.

June (Friday off, positions 2,4,5): 26 full days + 0.5 + 0.5 = 27. July (5 Fridays) → 26 + 1.5 = 27.5. Recalculated per month.

3. How late detection works

  • The UTC check-in is converted to the Office Timezone.
  • The session is determined; expected start = the session start (or the half-day-Friday start).
  • If the local check-in is later than start + Late Threshold, it is Is Late, with Late Minutes measured from the start time.
Example (Asia/Muscat): check-in stored 04:10 UTC = 08:10 Muscat; start 08:00 → 10 min late.

4. Deductions — Slab vs Hourly

Both apply only after grace and are 0 if waived.

Fixed (Slab-based)

Menu: Late Tracking → Deduction Slabs. The late minutes pick a slab; that slab's flat amount is deducted (matches when From ≤ late ≤ To; To = 0 = unlimited).

Hourly Wage

hourly_rate = monthly wage / working days in month / daily work hours deduction = hourly_rate × ceil(late_minutes / 60) (every started hour = 1 hour)
No-wage fallback: in Hourly mode, if an employee has no wage the hourly amount is 0, so it falls back to the matching slab. With no slab, the deduction is 0. Keep a basic slab as a safety net.

5. Grace & Late-Waiver

  • Grace: the first N late occurrences per month (per session) are free; a reason is still requested.
  • Enter Late Reason: on a late record, the employee clicks Enter Late Reason and types why.
  • Waiver: Late Tracking → Late Waiver Requests — pick the late record, give a reason, submit; a manager Approves (record → Waived, deduction 0) or Rejects.

6. Office-time display

Standard Odoo date columns show the viewer's timezone. The attendance lists/kanban/form, Late Waiver, Employee Report and the Late Attendance PDF show dedicated Office-Time values anchored to the Office Timezone, so everyone sees the same office time regardless of their settings.
